    DualBootPatcher - 介绍

    DualBootPatcher is an open-source app that allows multiple ROMs to be installed on a single Android device. It does its best to work with existing code and does not require explicit support from ROMs. There are currently 198 supported devices and their variations.

    官网：DualBootPatcher

    DualBootPatcher - 注意事项

    目前 小米手机 需要解锁 data
    不解锁副系统无法开机！
    一些人的提问:

    • 小米MAX刷了论坛大神提供的TWRP之后，显示要解锁DATA分区？
    • 一键去除system分区锁，解锁system
    • 关于data解锁求助
    • 新人请教system锁和data锁的问题
    • 最新开发版解锁方法！

    DualBootPatcher - 补丁教程

    需要使用多系统需要具备以下条件

        1. 有TWRP Recovery 并 可以单独刷入`img`文件（用来刷内核使用）
    2. 已经解锁BL或者破解
    3. 内存足够大（每刷入一个副系统不装软件大概占用`1.5G`请放心刷）
    4. 如果无法开机请在最底下寻找解决方法（按照我这样做保证系统不死,能变砖头的手机只有刷错别人的ROM情况下造成,或者刷了别人的底层,请放心刷机。本人几个手机是因为别人的ROM好看之后忍不住刷机 一个接一个的砖 9008 也没法救活只能换字库！知道会砖忍不住。现在乖了，一直没问题，大家吸取教训,有Recovery手机绝对不会变砖。）
    5. **这东西必须要ROOT才能用 - 我说的是完整的ROOT 不完整的没用 不是说有ROOT就行很多东西没法干**

    刷入副系统教程

    需要修改的位置

    • 部分机型的挂载点相同只需要修改Build.Prop
      • 例子：
        • LG G2
        • LG G2(D802)
        • LG G3 (F460也一样)
        • 这几款挂载表一样只需要修改Build.Prop
        • 你们也可以找一找相似的免去修改挂载点只需要修改Build.Prop

          如果没有相似的，请注意需要按照下面教程进行修改之后才能刷入 不修改刷进去无法开机,不相信的可以试一下,可以按照最后面的方法开机！

    • 打开补丁压缩包的dualboot/device.json文件夹，可以看到部分分区挂载点，包括by-name和mmcblk两种格式的，我们需要把这两种挂载点都改成我们机器的。

    如何查看by-name挂载点

    • 使用LINUX终端查看挂载点
      • 对于by-name挂载点，可以在终端中输入su，回车，输入mount，回车， 可能看起来比较乱，但是只要仔细找就不难发现挂载点,byname路径就是/dev/block/bootdevice/by-name，后面加上/分区就是该分区挂载点，例：/dev/block/bootdevice/by-name/boot；
    • 使用RE管理器查看挂载点
      • 用Root Explorer文件管理器进入/dev/block/platform/msm_sdcc.1/by-name
      • 找到boot、cache、system、data这几个挂载点 记录mmcblk0pxx（xx是各个分区的数字）
      • 把你打好补丁的ROMdualboot/device.json这个路径的改成你记录下的挂载点（改错刷进去也不会变砖最多就是无法开机，无法开机找到本文最后面的解决方法！）

    如何查看mmcblk挂载点

    • 对于mmcblk挂载点，可以使用Diskinfo.pro这个软件查看
      • 打开diskinfo pro
      • 右上角三个点→Settings
      • 勾选Expert Mode和它下面的Umounted Partitions
      • 返回，可以看到手机的各个分区
      • 点击在device.json中出现的分区，

    如何查看partition name

    • 查看partition name，例:mmcblk0p16
      • 那么该分区的挂载点就是/dev/block/mmcblk0p16,
      • 也就是/dev/block/加上partition name。
      • 按照获取到的挂载点修改device.json文件。

    如何修改机型验证

    • 刷机包就修改完毕了，但是dual boot还有一个它自己的机型验证
    • 上面我选择的是LG G2型号打的ROM补丁,就得跟这里一样才能刷进去通过验证
    • LG G2分区挂载 以及build.prop的值 - 搜索G2就能看到
      • 需要修改/system/build.prop
      • 【需要ROOT】修改工具 - BuildProp 点我酷安下载
      • 需要修改2个值！
        • ro.product.device 改为LG G2上面出现的
          • Device ID: lgg2
            • lgg2
        • ro.build.product 改为LG G2上面出现的
          • Codenames:
            • g2
            • d800
            • d801
            • ls980
            • vs980
            • f320
        • 如果你机器的build.prop中没有这两项，请自行添加
        • 我的设备是LG GPro2 - LG D838
          • 选的LG G2就改成这样

    DualBootPatcher - 无法开机解决方法

    如果刷入之后无法开机请进入 - Recovery 找到自带内存根目录 - MultiBoot 文件夹

    • primary文件夹里面存放着你的主系统内核 - boot
    • 其余文件夹则是你的副系统内核 如果无法开机从这里刷入你需要开机系统的内核即可！

    如果还是没法开机你可以选择直接刷一下你主系统的ROM 不需要双清 前提是ROM要一样 RR就只能刷RR 否则你只能双清或者四清
